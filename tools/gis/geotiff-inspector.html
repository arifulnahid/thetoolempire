GeoTIFF Inspector

Upload any GeoTIFF and instantly read its CRS, dimensions, band info, compression, geotransform, and bounding box. No server upload — everything runs in your browser.

Upload GeoTIFF
🔒 All processing happens in your browser using geotiff.js — no file data is sent to any server

What This Tool Reads

Everything stored in the GeoTIFF file header — without loading pixel data.

🗺️
CRS & EPSG code
Extracts the coordinate reference system from the GeoKeyDirectory tag. Shows EPSG code, Proj4 string (for common CRS), and the raw GeoAsciiParams CRS string. One-click copy.
📐
Geotransform
The 6-element affine matrix that maps pixel (col, row) to geographic (X, Y). Shows all six parameters with labels — origin X/Y, pixel width/height, rotation coefficients.
📦
Bounding box
Computes the spatial extent in the file's native CRS from the geotransform and image dimensions. Copy as minX,minY,maxX,maxY or JSON array.
🎨
Band info
Per-band: data type (uint8, int16, float32…), bits per sample, color interpretation (Red/Green/Blue/Gray), and no-data value from GDAL_NODATA tag.
🗜️
Compression & tiling
Detects LZW, Deflate, JPEG, ZSTD, WebP and other compression methods. Identifies tiled vs stripped layout. Flags Cloud-Optimised GeoTIFF (COG) structure.
🏷️
Embedded tags
Shows software name, creation date, image description, GDAL metadata XML, model tiepoint tag, and pixel scale tag from the TIFF file directory.

How to Use

1
Upload your GeoTIFF
Click the upload zone or drag-and-drop any .tif, .tiff, or .geotiff file. BigTIFF files and Cloud-Optimised GeoTIFFs are also supported. Only the file header is read — pixel data is not loaded.
2
Check the Overview tab
See image dimensions, band count, file size, compression method, and whether the file is a COG. The detected CRS name is shown here as a quick summary.
3
Inspect the CRS tab
Get the EPSG code, Proj4 string, and GeoAsciiParams for the embedded coordinate reference system. Copy the EPSG code or Proj4 to use in GDAL, pyproj, QGIS, or PostGIS.
4
Read geotransform & bbox
The Geotransform tab shows the affine matrix with labelled parameters and the computed bounding box. Copy the extent for use in GDAL warp, rasterio, or Python scripts.

Frequently Asked Questions

A GeoTIFF is a TIFF image file with embedded geographic metadata — coordinate reference system, bounding box, pixel resolution, and geotransform. It is the most common format for raster GIS data including satellite imagery, digital elevation models (DEMs), land-use maps, and climate datasets. The geo metadata is stored in special TIFF tags defined by the GeoTIFF specification.

The geotransform is a 6-element affine matrix that maps pixel coordinates (column, row) to geographic coordinates. The six values are: [top-left X, pixel width (W→E), row rotation, top-left Y, column rotation, pixel height (N→S, usually negative)]. For a north-up image with no rotation, elements 2 and 4 are zero. Formula: X = GT[0] + col × GT[1] + row × GT[2], Y = GT[3] + col × GT[4] + row × GT[5]. Used by GDAL and all GeoTIFF-aware tools to compute the real-world location of any pixel.

A Cloud-Optimised GeoTIFF (COG) is structured so that HTTP range requests can read only the part of the file needed for a given zoom level or spatial area, enabling efficient streaming from cloud storage (S3, GCS) without downloading the full file. COGs contain multiple internal overview levels (reduced-resolution copies) and use tiled rather than stripped organisation. The inspector flags COG structure when it detects multiple IFDs and a tiled layout.

Some GeoTIFF files are missing the GeoKeyDirectory tag that stores CRS information. This happens with older files, files exported without CRS metadata, or files that use GDAL sidecar files (.prj or .aux.xml) instead of embedded tags. The inspector can only read what is embedded in the file. To check sidecar CRS metadata, use gdalinfo yourfile.tif or QGIS Layer Properties.
